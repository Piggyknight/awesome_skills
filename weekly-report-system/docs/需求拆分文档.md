# 需求拆分文档

> 项目名称：周报LLM任务分类系统
> 创建时间：2026-03-08
> 项目经理：Claw (project-manager)

## 1. 项目概述

在现有周报系统基础上，增加基于LLM的智能任务分类功能，将周报中的任务按照技术领域自动分类到：内存相关、资源相关、鸿蒙支持、CI相关、音频相关等类别，提高周报的结构化程度和可读性。

**核心功能**：
1. **实时分类**：在日报收集和周报生成时自动分类
2. **历史周报分类**：对已生成的team周报批量分类，生成分组周报 🆕

## 2. 功能需求

### 2.1 任务分类规则配置模块
- **描述**：读取和解析分类规则配置文件（task_category.md），将自然语言描述转换为结构化的分类规则
- **优先级**：高
- **技术要点**：
  - 解析Markdown格式的分类规则文档
  - 提取每个分类的关键词、描述、边界条件
  - 支持分类规则的动态更新
- **验收标准**：
  - [ ] 能正确解析task_category.md中的5个分类
  - [ ] 提取的关键词准确率≥90%
  - [ ] 支持配置文件热更新

### 2.2 LLM分类服务模块
- **描述**：利用OpenClaw GLM模型（zai/glm-5）对任务进行智能分类，支持单任务分类和批量分类
- **优先级**：高
- **技术要点**：
  - 集成现有llm_service.py模块
  - 设计合适的prompt模板，将分类规则注入prompt
  - 处理LLM返回结果，解析分类标签
  - 支持多分类（一个任务可能属于多个类别）
- **验收标准**：
  - [ ] 单任务分类准确率≥85%
  - [ ] 支持批量分类（≥100条任务）
  - [ ] LLM调用失败有重试机制
  - [ ] 响应时间<2秒/任务

### 2.3 任务分类集成模块
- **描述**：将分类功能集成到现有的日报收集和周报生成流程中
- **优先级**：高
- **技术要点**：
  - 在daily_collector.py中增加分类步骤
  - 在weekly_generator.py中增加分类统计功能
  - 修改数据结构，增加category字段
  - 保持向后兼容，未分类任务标记为"未分类"
- **验收标准**：
  - [ ] 日报收集时自动分类
  - [ ] 周报中包含分类统计信息
  - [ ] 现有功能不受影响

### 2.4 分类结果展示模块
- **描述**：在周报输出中增加分类展示，支持按类别查看任务统计
- **优先级**：中
- **技术要点**：
  - 修改周报Markdown模板，增加分类统计章节
  - 支持按类别分组展示任务
  - 生成分类统计图表（饼图/柱状图）
  - 支持多格式导出（Markdown/HTML）
- **验收标准**：
  - [ ] 周报包含"分类统计"章节
  - [ ] 每个类别显示任务数量和占比
  - [ ] 支持图表可视化

### 2.5 历史周报批量分类模块 🆕
- **描述**：对已生成的team周报进行批量分类处理，生成按技术领域分组的分类周报
- **优先级**：高
- **技术要点**：
  - 读取已存在的team周报Markdown文件
  - 解析周报中的所有任务项
  - 批量调用LLM进行任务分类
  - 生成分类版周报（按类别组织任务）
  - 支持指定周次范围（如：2026-W01 到 2026-W10）
  - 输出文件命名：`team_YYYY-WXX_classified.md`
- **验收标准**：
  - [ ] 支持读取现有team周报文件
  - [ ] 批量分类准确率≥85%
  - [ ] 生成的分类周报结构清晰
  - [ ] 包含分类统计摘要
  - [ ] 处理100条任务耗时<3分钟

### 2.6 分类准确率优化模块
- **描述**：提供人工反馈机制，持续优化分类准确率
- **优先级**：低
- **技术要点**：
  - 记录分类结果和人工修正
  - 分析常见错误模式
  - 优化prompt模板
  - 建立分类测试集
- **验收标准**：
  - [ ] 支持人工修正分类
  - [ ] 记录修正历史
  - [ ] 优化后准确率提升≥5%

## 3. 非功能需求

- **性能**：批量分类100条任务耗时<3分钟
- **准确性**：初始分类准确率≥85%，优化后≥90%
- **可扩展性**：支持新增分类类别，无需修改核心代码
- **容错性**：LLM服务异常时降级为关键词匹配
- **可维护性**：分类规则配置化，代码模块化

## 4. 约束条件

- 必须使用现有技术栈（Python 3.8+, OpenClaw GLM模型）
- 不得影响现有周报系统的稳定性
- 分类配置文件格式保持Markdown
- LLM调用需要控制成本（避免过度调用）

```

│   ├── task_classifier.py (新增) - LLM分类核心
```

### 5.2 数据流

**流程1：实时分类流程（日报→周报）**
```
日报输入 → 解析任务 → LLM分类 → 存储分类结果 → 周报生成 → 分类统计展示
```

**流程2：历史周报分类流程（新增）🆕**
```
已生成周报 → 读取文件 → 提取任务列表 → 批量LLM分类 → 生成分类版周报 → 统计展示
```
