# 需求拆分文档：日报解析增强

## 1. 背景与问题

### 1.1 问题描述

当前日报解析器 (`src/core/parser.py`) 无法区分"任务"和"描述"。所有行都经过 `strip()` 处理，导致缩进信息丢失，带缩进的内容被错误地当作独立任务。

### 1.2 案例分析

**用户输入：**
```
今 搞打包内存高的问题
    死锁的解决方案测试通过, 不再死锁
    效果
        BuildClient从40+GB降到10+GB
        平台资源生成从30+GB降到10+GB
```

**当前错误输出：**
- 任务1: "搞打包内存高的问题"
- 任务2: "死锁的解决方案测试通过, 不再死锁"  ← 错误！这应该是描述
- 任务3: "效果"  ← 错误！
- 任务4: "BuildClient从40+GB降到10+GB"  ← 错误！
- 任务5: "平台资源生成从30+GB降到10+GB"  ← 错误！

**期望正确输出：**
- 任务1:
  - title: "搞打包内存高的问题"
  - description:
    - "死锁的解决方案测试通过, 不再死锁"
    - "效果":
      - "BuildClient从40+GB降到10+GB"
      - "平台资源生成从30+GB降到10+GB"

## 4. 开发任务拆分

### 任务1：修改解析器核心逻辑

**文件**: `src/core/parser.py`

**改动点**:

1. **保留缩进信息**：
   ```python
   # 之前
   line = line.strip()
   
   # 之后
   original_line = line
   stripped_line = line.strip()
   indent_level = len(original_line) - len(original_line.lstrip())
   ```

2. **新增辅助函数** - 计算缩进级别：
   ```python
   def get_indent_level(line: str) -> int:
       """计算行的缩进级别（4空格=1级）"""
       stripped = line.lstrip()
       if not stripped:
           return 0
       spaces = len(line) - len(stripped)
       return spaces // 4
   ```

3. **修改任务解析逻辑**：
   ```python
   def parse_task_with_description(lines: List[str], start_idx: int) -> Tuple[Dict, int]:
       """
       解析任务及其描述
       
       Returns:
           (task_dict, next_index)
       """
       task = {"title": "", "description": []}
       
       # 当前行为任务标题
       task["title"] = lines[start_idx].strip()
       
       # 向后查找缩进内容作为描述
       idx = start_idx + 1
       while idx < len(lines):
           line = lines[idx]
           indent = get_indent_level(line)
           
           if indent == 0:
               # 无缩进，任务结束
               break
           
           # 有缩进，作为描述添加
           description_item = parse_description_item(line, indent)
           task["description"].append(description_item)
           idx += 1
       
       return task, idx
   ```

### 任务2：实现描述项解析

**新增函数**:

```python
def parse_description_item(line: str, indent: int, lines: List[str], idx: int) -> Tuple[Union[str, Dict], int]:
    """
    解析单个描述项
    
    Args:
        line: 当前行
        indent: 缩进级别
        lines: 所有行
        idx: 当前索引
    
    Returns:
        (description_item, next_index)
    """
    content = line.strip()
    
    # 检查是否有子项（下一行缩进更深）
    if idx + 1 < len(lines):
        next_indent = get_indent_level(lines[idx + 1])
        if next_indent > indent:
            # 当前项是标签，后续是子项
            items = []
            new_idx = idx + 1
            while new_idx < len(lines):
                next_line = lines[new_idx]
                next_i = get_indent_level(next_line)
                if next_i <= indent:
                    break
                if next_i == indent + 1:
                    items.append(next_line.strip())
                new_idx += 1
            
            return {"label": content, "items": items}, new_idx
    
    # 简单文本描述
    return content, idx + 1
```

**预计工作量**: 1小时

```python
def _convert_to_markdown(self, report: Dict) -> str:
    """转换日报为markdown格式（支持任务描述）"""
    lines = [f"# 日报 - {report.get('date', '')}", ""]
    
    for member, data in report.get("members", {}).items():
        lines.append(f"## {member}")
        lines.append("")
        
        # 今日任务
        lines.append("### 今日完成")
        for task in data.get("today", []):
            if isinstance(task, str):
                # 旧格式兼容
                lines.append(f"- {task}")
            else:
                # 新格式
                lines.append(f"- {task['title']}")
                for desc in task.get("description", []):
                    if isinstance(desc, str):
                        lines.append(f"    {desc}")
                    else:
                        lines.append(f"    {desc['label']}")
                        for item in desc.get("items", []):
                            lines.append(f"        {item}")
        lines.append("")
        
        # 明日计划（类似处理）
        ...
    
    return "\n".join(lines)
```


```
0          任务标题（无缩进）
1          任务描述/详情（4空格）
2          描述子项（8空格）
3+         更深层级（12+空格，暂不支持）
```

```

    效果           ← 级别1（作为标签）

        具体效果2  ← 级别2
```
