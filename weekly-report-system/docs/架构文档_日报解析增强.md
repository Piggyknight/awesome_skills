## 5. 核心算法设计

### 5.1 缩进级别计算算法

#### **算法1: 基础缩进计算**

```python
def calculate_indent_level(line: str, spaces_per_level: int = 4) -> int:
    """
    计算行的缩进级别
    
    算法步骤:
    1. 去除左侧空格，获取实际内容
    2. 计算空格数 = 原始长度 - 去除空格后长度
    3. 缩进级别 = 空格数 / 每级空格数
    
    时间复杂度: O(n) - n为行的长度
    空间复杂度: O(1)
    """
    stripped = line.lstrip()
    if not stripped:
        return 0
    
    spaces = len(line) - len(stripped)
    return spaces // spaces_per_level
```

#### **算法2: 自动检测缩进风格**

```python
def detect_indent_style(lines: List[str]) -> int:
    """
    检测文本使用的缩进风格（2空格或4空格）
    
    算法步骤:
    1. 收集所有缩进行的空间数
    2. 计算空间数的GCD（最大公约数）
    3. 如果GCD是2，则使用2空格缩进
    4. 否则默认使用4空格缩进
    
    时间复杂度: O(n*m) - n为行数，m为平均行长
    空间复杂度: O(k) - k为不同缩进值的数量
    """
    indent_values = []
    
    for line in lines:
        stripped = line.lstrip()
        if stripped and stripped != line:
            spaces = len(line) - len(stripped)
            indent_values.append(spaces)
    
    if not indent_values:
        return 4  # 默认4空格
    
    # 计算GCD
    from math import gcd
    from functools import reduce
    
    gcd_value = reduce(gcd, indent_values)
    
    # 判断缩进风格
    if gcd_value == 2:
        return 2
    else:
        return 4
```

### 5.2 层级关系解析算法

#### **算法3: 任务与描述归属判断**

```python
def parse_task_with_description(lines: List[str], start_idx: int) -> Tuple[Dict, int]:
    """
    解析任务及其描述
    
    算法步骤:
    1. 提取任务标题（start_idx位置，缩进级别0）
    2. 向后遍历后续行
    3. 如果缩进级别 > 0，则为描述，添加到description列表
    4. 如果缩进级别 = 0，则任务结束，返回
    5. 递归处理嵌套描述
    
    时间复杂度: O(m) - m为描述的行数
    空间复杂度: O(d) - d为描述的深度
    """
    task = {
        "title": lines[start_idx].strip(),
        "description": []
    }
    
    idx = start_idx + 1
    while idx < len(lines):
        line = lines[idx]
        indent = calculate_indent_level(line)
        
        # 缩进级别0，任务结束
        if indent == 0:
            break
        
        # 解析描述项
        desc_item, idx = parse_description_item(lines, idx, indent)
        task["description"].append(desc_item)
    
    return task, idx
```

#### **算法4: 描述项解析（支持嵌套）**

```python
def parse_description_item(
    lines: List[str],
    idx: int,
    current_indent: int
) -> Tuple[Union[str, Dict], int]:
    """
    解析单个描述项（支持嵌套）
    
    算法步骤:
    1. 提取当前行内容
    2. 检查下一行是否缩进更深
    3. 如果是，则当前行是标签，构建分组对象
    4. 递归收集所有子项（缩进级别 = current_indent + 1）
    5. 如果否，则返回简单字符串
    
    时间复杂度: O(m) - m为子项的行数
    空间复杂度: O(k) - k为子项的数量
    """
    content = lines[idx].strip()
    
    # 检查是否有子项
    if idx + 1 < len(lines):
        next_indent = calculate_indent_level(lines[idx + 1])
        
        if next_indent > current_indent:
            # 当前项是标签，后续是子项
            group = {
                "label": content,
                "items": []
            }
            
            idx += 1
            child_indent = current_indent + 1
            
            # 收集所有子项
            while idx < len(lines):
                line_indent = calculate_indent_level(lines[idx])
                
                # 缩进减少或为0，子项结束
                if line_indent <= current_indent:
                    break
                
                # 只收集直接子项（缩进级别 = current_indent + 1）
                if line_indent == child_indent:
                    group["items"].append(lines[idx].strip())
                
                idx += 1
            
            return group, idx
    
    # 简单文本描述
    return content, idx + 1
```

### 5.3 算法复杂度总结

| 算法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|------------|------------|------|
| 缩进级别计算 | O(n) | O(1) | n为行长度 |
| 缩进风格检测 | O(n*m) | O(k) | n为行数，m为平均行长，k为不同缩进值 |
| 任务解析 | O(m) | O(d) | m为描述行数，d为嵌套深度 |
| 描述项解析 | O(m) | O(k) | m为子项行数，k为子项数量 |

## 6. 扩展性设计

### 6.1 支持更深层级的嵌套

**当前设计**: 支持最多2层嵌套（任务 -> 描述 -> 子项）

**扩展方案**: 将 `DescriptionItem` 改为递归结构

```python
# 当前设计
class DescriptionItem(TypedDict):
    label: str
    items: List[str]

# 扩展设计（支持无限层级）
DescriptionItem = TypedDict('DescriptionItem', {
    'label': str,
    'items': List[Union[str, 'DescriptionItem']]  # 递归定义
})
```
