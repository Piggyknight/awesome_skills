# 项目经理 Agent 工作流参考

## 核心原则：上下文隔离 🆕

**每个 Agent 都是独立执行的，不共享对话历史。**

### 上下文隔离流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    项目经理 (协调者)                              │
│                                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐            │
│  │需求文档  │  │架构文档  │  │代码文件  │  │测试报告  │            │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘            │
│       │            │            │            │                  │
└───────┼────────────┼────────────┼────────────┼──────────────────┘
        │            │            │            │
        │            │            │            │
        ▼            ▼            ▼            ▼
   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
   │ 架构师   │  │ 开发者   │  │ Debug   │  │   QA    │
   │         │  │         │  │         │  │         │
   │ 输入:   │  │ 输入:   │  │ 输入:   │  │ 输入:   │
   │ 需求文档 │  │ 架构文档 │  │ 代码文件 │  │ 代码+标准│
   │         │  │         │  │         │  │         │
   │ 输出:   │  │ 输出:   │  │ 输出:   │  │ 输出:   │
   │ 架构文档 │  │ 代码文件 │  │ 带日志   │  │ 测试报告 │
   └─────────┘  └─────────┘  └─────────┘  └─────────┘
        │            │            │            │
        └────────────┴────────────┴────────────┘
                           │
                           ▼
                    返回给项目经理
```

### 上下文隔离规则

| 规则 | 说明 |
|------|------|
| **文档是唯一接口** | Agent 之间通过文档通信，不通过对话 |
| **只读相关文档** | 每个 Agent 只接收完成任务所需的文档 |
| **无历史记忆** | 每个 Agent 会话开始时上下文为空 |
| **输出到文档** | 结果必须写入文档，供下游 Agent 使用 |

### Agent 类型与任务分配 🆕

| Agent 类型 | 适用任务 | 标识符 |
|-----------|---------|--------|
| **C++ 游戏前端** | Unreal Engine C++ 代码 | `developer` |
| **Web 前端** | React/TypeScript 代码 | `frontend-developer` |
| **Python 后端** | RESTful API / 数据库 | `backend-developer` |
| **调试** | 嵌入日志 | `debug` |
| **测试** | 验收测试 | `qa` |
| **Git** | 版本控制 | `git` |

### 任务类型判断

```
任务包含关键词:
├─ "Unreal"、"UE"、"C++"、"游戏" → developer (C++ 游戏前端)
├─ "React"、"Web"、"前端页面"、"UI组件" → frontend-developer (Web 前端)
├─ "API"、"后端"、"数据库"、"服务端" → backend-developer (Python 后端)
├─ "调试"、"日志" → debug
├─ "测试"、"验收" → qa
└─ "Git"、"提交"、"推送" → git
```

## 完整工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                          用户需求                                │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐

│  │  3. 拆分功能模块                                         │    │
│  │  4. 定义验收标准                                         │    │
│  │  5. 输出: docs/需求拆分文档.md                           │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
                    通知架构师 Agent
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  (架构师 Agent 工作...)                                         │
│  输出: 架构文档.md + 开发任务列表.md                             │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  阶段2: 进度管理                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  1. 读取开发任务列表.md                                   │    │
│  │  2. 创建项目开发进度报告.md                               │    │
│  │  3. 按优先级串行分配任务                                   │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
                                │
                ┌───────────────┴───────────────┐
                │      循环处理每个任务           │
                └───────────────┬───────────────┘
                                │
                                ▼
                ┌───────────────────────────────┐
                │   分配任务给开发者 Agent       │
                └───────────────┬───────────────┘
                                │
                                ▼
                ┌───────────────────────────────┐
                │   等待 QA Agent 结果           │
                └───────────────┬───────────────┘
                                │
                ┌───────────────┴───────────────┐
                │                               │
            通过 ▼                         失败 ▼
    ┌───────────────────┐        ┌───────────────────┐
    │ 1. 更新进度报告    │        │ 1. 记录失败次数    │
    │ 2. 通知 Git Agent  │        │ 2. 检查是否 ≥5次   │
    │ 3. 分配下一任务    │        │ 3. <5次: 返回修复  │
    └───────────────────┘        │ 4. ≥5次: 暂停项目  │
                                  └───────────────────┘
```

## 状态机

### 任务状态

```
⏳ 待开始 → 🔄 进行中 → ✅ 已完成
                │
                ▼
            ⏸️ 阻塞中
                │
                ▼
            🔄 进行中 (重试)
```

### 项目状态

```
进行中 ←→ 已暂停
    │
    ▼
  已完成
```

## 错误处理矩阵

| 错误类型 | 检测条件 | 处理动作 |
|---------|---------|---------|
| 任务失败 | QA返回失败 | 重试 (<5次) 或暂停 (≥5次) |
| 文档缺失 | 读取文件失败 | 提示上游 Agent 补充 |
| 架构变更 | 架构师通知 | 重新评估任务列表 |
| 用户中断 | 用户明确要求 | 暂停并保存状态 |

### QA 流程详解

```
┌─────────────────────────────────────────────────────────────┐
│                    QA 完整验收流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 生成测试                                            │
│  ├─ 读取架构文档验收标准                                      │
│  └─ 生成单元测试文件                                         │
│         ↓                                                   │
│  Step 2: 编译项目                                            │
│  ├─ 运行 UE 编译命令                                         │
│  ├─ 成功 → 继续                                              │
│  └─ 失败 → 返回编译错误                                       │
│         ↓                                                   │
│  Step 3: 运行测试                                            │
│  ├─ 运行 UE 自动化测试                                        │
│  ├─ 解析测试结果                                             │
│  └─ 计算覆盖率                                               │
│         ↓                                                   │
│  Step 4: 输出结果                                            │
│  ├─ 全部通过 → 测试报告 → Git Agent                          │
│  └─ 有失败 → 错误文档 → 返回开发者                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 上下文隔离实现

### 使用 sessions_spawn

```python
def spawn_agent_with_isolated_context(
    agent_id: str,
    task: str,
    input_files: list,
    timeout_seconds: int = 1800
):
    """创建隔离上下文的子会话"""

    # 读取输入文件内容
    context_content = ""
    for file_path in input_files:
        with open(file_path, 'r', encoding='utf-8') as f:
            context_content += f"\n\n--- {file_path} ---\n\n"
            context_content += f.read()

    # 构造任务提示
    full_task = f"""
{task}

---

## 输入文档

{context_content}

---

请根据以上文档完成任务。完成后将结果写入指定文件。
"""

    # 创建子会话
    return sessions_spawn(
        agent_id=agent_id,
        task=full_task,
        timeout_seconds=timeout_seconds,
        cleanup="delete"  # 完成后删除会话
    )
```
