# 周报生成技能

你是周报生成专家，负责从一周的日报中生成团队周报和个人周报。

## 核心职责

| 输入 | 输出 | 下游 |
|------|------|------|
| 一周的日报文件 | 团队周报 + 个人周报 | 邮件发送 |

---

## 使用场景

1. **生成指定周的周报**: 根据周标识生成周报
2. **自动发送周报邮件**: 生成后自动发送给管理员和成员
3. **查看周报统计**: 查看本周任务统计信息

---


## 周报类型

### 1. 团队周报

**文件名**: `team_weekly_YYYYMMDD.md`

**内容**:
- 汇总所有成员的"今日完成"任务
- 去除重复任务
- 使用LLM润色（可选）
- 按分类组织（如：功能开发、BUG修复、性能优化等）

**示例**:
```markdown
# 团队周报汇总

**周期**: 2026-03-02 ~ 2026-03-07 (第10周)

## ● 本周完成的工作:

- 内存相关
  - mmap文件压缩：3900+动画mmap文件压缩至40个文件...
  - #104591 编辑器下脏数据功能罗列...
  - 打包内存优化：BuildClient从40+GB降到10+GB...

- 资源系统
  - 多语言hotfix热更已测试完毕...
  - hotfix表格改成json文件通过...

- 鸿蒙支持
  - 音频播放测试：增加测试代码...
  - 鸿蒙项目打开慢性能调查...
```

### 2. 个人周报

**文件名**: `soc_weekly_起始_结束_成员ID.md`

**示例**: `soc_weekly_20260302_20260307_HLQ.md`

**内容**:
- 该成员本周所有"今日完成"任务
- 去重后的任务列表
- 保留原始描述

**示例**:
```markdown
# HLQ 周报

**周期**: 2026-03-02 ~ 2026-03-07

## ● 本周完成的工作:

- #104591 编辑器下脏数据功能罗列
- 鸿蒙 - Ab打包疏通
- 下载中心 - 地图分包标签规则
- 组内会议
- trunk长线跑测崩溃

---
*此周报由自动化系统生成*
```

## 处理逻辑

### 1. 读取日报

从 `~/Documents/Report/daily/` 目录读取一周内的所有日报:
- 周一到周日（7天）
- 自动识别日期范围

### 2. 任务收集

- 提取所有"今日完成"任务
- 按成员分组
- 统计任务数量

### 3. 任务去重

使用相似度算法:
- 计算任务相似度 (0-1)
- 相似度 > 0.7 的任务合并
- 保留最长的任务描述

### 4. LLM润色

调用OpenClaw GLM模型 (zai/glm-5):
- 精简语句
- 突出重点
- 保持技术细节
- 分类组织

### 5. 生成周报

- 团队周报 → `data/weekly/team/`
- 个人周报 → `data/weekly/members/`

### 6. Git提交

自动提交到Git仓库:
- 提交信息: `生成周报 YYYY-WXX`

### 7. 邮件发送

- 团队周报 → 发送给管理员 (Kai)
- 个人周报 → 发送给对应成员

## 集成说明

本技能与以下系统模块集成:

1. **周报生成模块** (`src/modules/weekly_generator.py`)
   - 核心生成逻辑

2. **日报收集模块** (`src/modules/daily_collector.py`)

3. **LLM服务** (`src/core/llm_service.py`)
   - GLM模型调用
   - 文本润色

4. **任务去重** (`src/core/task_dedup.py`)
   - 相似度计算
   - 任务合并

5. **邮件发送** (`src/core/email_sender.py`)
   - 批量发送邮件
   - 个性化内容

6. **Git操作** (`src/core/git_helper.py`)
   - 自动提交
   - 版本控制

## 参数说明

### --week
- **说明**: 周标识
- **格式**: `YYYY-WXX` (如: 2026-W10)
- **默认**: 当前周

### --start-date
- **说明**: 起始日期
- **格式**: `YYYY-MM-DD` 或 `YYYYMMDD`
- **默认**: 本周一

### --send-email
- **说明**: 生成后发送邮件
- **默认**: 不发送

### --team-only
- **说明**: 只生成团队周报
- **默认**: 生成两种周报

### --members-only
- **说明**: 只生成个人周报
- **默认**: 生成两种周报

### --no-llm
- **说明**: 不使用LLM润色
- **默认**: 使用LLM润色

## 错误处理

### 常见错误

1. **没有日报数据**
   - 错误: "本周没有任何日报"
   - 解决: 先收集日报

2. **LLM调用失败**
   - 错误: "LLM润色失败"
   - 解决: 使用 `--no-llm` 跳过润色

3. **邮件发送失败**
   - 错误: "邮箱地址未配置"
   - 解决: 检查 `members.json`

4. **Git提交失败**
   - 错误: "Git操作失败"
   - 解决: 检查Git仓库状态
